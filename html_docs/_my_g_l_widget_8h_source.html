166  /*
167  * normally OpenGL is for the C language (i.e. no objects). Thus all
168  * Opengl functions can be called directly provided we include
169  * the header files.
170 
171  * With newer modern versions of opengl, this gets messy. This is
172  * because (a) we must have the correct version of the header file that
173  * has all the functions in a particular OpenGL version. (b) this correct
174  * file must be included at every place where we are using OpenGL
175  * functions. For this simple application, that ends up being here,
176  * View, ObjectInstance classes. To change the OpenGL version for any
177  * reasons, all these files must be edited.
178  *
179  * Therefore we wrap all the OpenGL functionality in an object, which
180  * we pass around. Qt has several wrapper classes, one for each version
181  * of OpenGL (and even OpenGL ES). In order to make our code even
182  * more independent of Qt, we create a new class OpenGLFunctions. For
183  * now we make it extend the QOpenGLFunctions_3_3_Core, which
184  * encapsulates all the OpenGL 3.3 core functionality.
185  *
186  * To change the version or even to use with a different library,
187  * simply change the OpenGLFunctions class to have the correct parent
188  */
