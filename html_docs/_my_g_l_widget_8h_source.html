120  /*
121  * normally OpenGL is for the C language (i.e. no objects). Thus all
122  * Opengl functions can be called directly provided we include
123  * the header files.
124 
125  * With newer modern versions of opengl, this gets messy. This is
126  * because (a) we must have the correct version of the header file that
127  * has all the functions in a particular OpenGL version. (b) this correct
128  * file must be included at every place where we are using OpenGL
129  * functions. For this simple application, that ends up being here,
130  * View, ObjectInstance classes. To change the OpenGL version for any
131  * reasons, all these files must be edited.
132  *
133  * Therefore we wrap all the OpenGL functionality in an object, which
134  * we pass around. Qt has several wrapper classes, one for each version
135  * of OpenGL (and even OpenGL ES). In order to make our code even
136  * more independent of Qt, we create a new class OpenGLFunctions. For
137  * now we make it extend the QOpenGLFunctions_3_3_Core, which
138  * encapsulates all the OpenGL 3.3 core functionality.
139  *
140  * To change the version or even to use with a different library,
141  * simply change the OpenGLFunctions class to have the correct parent
142  */
